# Holdem Noter

> 홀덤 플레이어를 위한 **핸드 기록 · 복기 · 상대 분석 노트 앱**
> 실제 플레이 중 느낀 복기 비효율 문제를 해결하기 위해 기획하고, **APK 배포까지 완료한 실사용 프로젝트**입니다.

---

## 📌 프로젝트 소개

홀덤을 플레이하면서 가장 불편했던 점은 다음과 같았습니다.

* 중요한 핸드를 바로 기록하기 어렵다
* 상대 플레이어 성향을 체계적으로 저장하기 힘들다
* 특정 상황(BTN, 3bet pot, bluff catch 등) 복기가 어렵다
* 시간이 지나면 핸드 히스토리를 다시 찾기 힘들다

이 문제를 해결하기 위해 **핸드 상황을 빠르게 기록하고, 태그 기반으로 검색하며, 상대 성향까지 정리할 수 있는 모바일 노트 앱**을 직접 개발했습니다.

실제 APK로 배포 후 직접 사용하며 기능을 개선했습니다.

---

## ✨ 주요 기능

### 📝 핸드 기록

* 프리플랍 / 플랍 / 턴 / 리버 액션 저장
* 포지션 정보 저장 (UTG, MP, CO, BTN, SB, BB)
* 보드 텍스처 기록
* 쇼다운 결과 기록

### 🧠 상대 분석 노트

* 플레이어별 성향 메모
* 블러프 빈도 / 3bet 성향 기록
* VPIP / PFR 스타일 태그화
* 플레이 스타일 분류 (TAG / LAG / Nit / Calling Station)

### 🔍 검색 및 필터링

* 포지션별 검색
* 특정 핸드 검색
* 상대별 핸드 검색
* 태그 기반 복합 필터
* 세션별 기록 조회

### 📦 배포

* APK 빌드 및 실제 디바이스 배포 완료
* 실사용 기반 피드백 반영

## 🏗 아키텍처 및 설계 포인트

### 1) 도메인 중심 데이터 구조 설계

핸드 복기를 빠르게 하기 위해 다음과 같이 데이터 구조를 분리했습니다.

* `hands` : 핸드 기본 정보
* `players` : 상대 플레이어 정보
* `notes` : 성향 메모
* `tags` : 상황 분류 태그
* `sessions` : 날짜별 세션 기록

백엔드 확장 시 그대로 **RDB(PostgreSQL/MySQL) 테이블 설계로 이관 가능한 구조**를 고려했습니다.

### 2) 빠른 검색을 위한 태그 기반 구조

실전 복기에서는 아래와 같은 검색이 중요합니다.

> BTN에서 AQo로 3bet 당한 핸드만 보기
> 특정 상대의 river bluff catch 상황만 보기

이를 위해 **태그 기반 필터링 구조**를 적용해 검색 경험을 개선했습니다.

### 3) 실사용 중심 UX

직접 플레이어 입장에서 사용하기 때문에:

* 빠른 입력
* 최소 탭 이동
* 즉시 검색
* 세션 기준 복기

위 흐름을 가장 중요하게 설계했습니다.

## ⚡ 트러블슈팅

### 문제 1. 핸드가 많아질수록 검색 속도 저하

기록 수가 늘어나면서 태그 검색 속도가 느려졌습니다.

### 해결

* 태그 인덱싱 구조 개선
* 메모리 캐싱
* 불필요한 전체 렌더링 최소화
* 리스트 렌더 최적화

### 결과

대량 핸드 기록에서도 특정 상황 검색 속도를 개선했습니다.
